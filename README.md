# ConcertGo - 演唱会抢票系统

## How to Run

### Docker Compose 一键启动（推荐）

```bash
docker-compose up --build -d
```

等待所有服务启动完成（约2-3分钟），然后访问：

- 用户端：http://localhost:8081
- 管理后台：http://localhost:8082
- 后端 API：http://localhost:8080
- RabbitMQ 管理界面：http://localhost:15672 (guest/guest)

### 本地开发启动

1. 启动 MySQL 8.0、Redis、RabbitMQ
2. 执行 `backend/src/main/resources/schema.sql` 建表
3. 后端启动：
```bash
cd backend
mvn spring-boot:run
```
4. 前端直接用浏览器打开 `frontend-user/index.html` 和 `frontend-admin/index.html`

### 停止服务

```bash
docker-compose down
# 如需清除数据
docker-compose down -v
```

## 功能使用说明

### 记住密码功能

用户端登录时支持"记住密码（7天）"功能，实现 7 天内免登录访问。

#### 使用方法

1. **首次登录**
   - 访问用户端 http://localhost:8081
   - 点击右上角"登录"按钮
   - 输入用户名和密码（如 `zhangsan` / `User@1234`）
   - **勾选"记住密码（7天）"复选框**
   - 点击"登录"

2. **自动免登录**
   - 关闭浏览器或标签页
   - 7 天内再次访问 http://localhost:8081
   - 系统自动恢复登录状态，无需重新输入密码

3. **手动退出**
   - 点击右上角"退出"按钮
   - 清除所有登录信息（包括记住密码的令牌）
   - 下次访问需要重新登录

#### 技术实现

- **后端**：登录时签发 7 天有效期的长令牌（JWT），普通会话令牌仍为 2 小时
- **前端**：使用 Web Crypto API 的 AES-256-GCM 加密算法存储长令牌
- **安全性**：
  - PBKDF2 密钥派生（10万次迭代 + SHA-256）
  - 随机 IV（初始化向量），每次加密结果不同
  - 令牌以密文形式存储在 localStorage，无法直接读取
  - 页面加载时验证令牌有效性，失效自动清除

#### 注意事项

- 记住密码功能仅在用户端（http://localhost:8081）可用
- 管理后台（http://localhost:8082）不支持记住密码
- 清除浏览器缓存会删除记住的登录信息
- 7 天后令牌自动过期，需要重新登录

## 题目内容

开发一个基于  Spring Boot（Maven 构建）+ 原生 HTML/CSS/JavaScript  的演唱会抢票系统，核心支撑高并发抢票、实时库存管理、 用户登录注册、订单全流程管理等能力，1. 用户管理模块 
注册： 
表单验证（用户名唯一、邮箱格式、手机号正则、密码强度） 
密码 BCrypt 加密存储，禁止明文 / MD5 简单加密 
注册后发送激活通知（模拟，无真实邮件 / 短信，仅前端提示） 
登录注册： 
生成（包含用户 ID、角色，过期时间 2 小时） 
登录限流（5 分钟内错误 5 次锁定 10 分钟） 
记住密码（前端 localStorage 存储加密后的令牌，有效期 7 天） 
信息管理： 
个人信息（用户名、邮箱、手机号、头像）修改 / 查询 
用户状态（正常 / 禁用）后台管理，禁用用户无法登录 
权限控制： 
普通用户：抢票、下单、查订单 
管理员：演唱会 / 票务管理、订单审核、用户状态管理 
2. 演唱会管理模块（仅管理员） 
信息维护： 
增删改查演唱会信息（标题、描述、艺人、场馆、城市、演出日期 / 时间、海报 URL、状态） 
状态枚举：未开始 / 售票中 / 已结束 / 已下架，仅 “售票中” 可抢票 
筛选查询： 
按城市、演出日期、状态筛选，按日期倒序排序 
支持演唱会名称模糊搜索 
3. 票务管理模块（核心高并发） 
票务配置： 
关联演唱会，配置票档（普通票 / VIP 票）、价格、总库存、可售库存 
库存初始化同步至 Redis（MySQL+Redis 双存储，保证最终一致） 
抢票核心： 
接口限流（Redis 令牌桶，单用户 / IP 10 次 / 分钟） 
库存预扣减（Redis 原子操作，防止超卖，预扣减有效期 15 分钟） 
超卖防护：Redis 库存扣减后，异步校验 MySQL 库存，不一致则回滚 
库存释放： 
未支付订单 15 分钟后，自动释放 Redis+MySQL 库存（RabbitMQ 延迟队列） 
手动取消订单，即时释放库存 
4. 订单管理模块 
订单流程： 
创建：抢票成功后生成订单（待支付状态），关联用户、演唱会、票档、数量、总价 
支付：模拟支付接口（前端点击 “支付” 后修改订单状态为 “已支付”） 
取消：待支付状态可取消，已支付状态支持退款（修改为 “已退款”） 
完结：演出结束后，已支付订单自动标记为 “已完成” 
订单查询： 
普通用户：仅查自己的订单，按创建时间倒序 
管理员：查所有订单，支持按订单号、用户、演唱会、状态筛选 
订单日志： 
记录订单状态变更（创建 / 支付 / 取消 / 退款），包含操作时间、操作人 
5. 通知模块 
实时通知： 
WebSocket 推送（抢票结果、订单状态变更、系统公告） 
前端未读通知小红点提示，支持标记已读 / 全部已读 
通知类型： 
系统公告（管理员发布，全员可见） 
个人通知（订单 / 抢票相关，仅本人可见） 
通知存储： 
MySQL 存储通知记录，支持分页查询历史通知

### 功能模块

1. **用户管理模块**：注册（表单验证、BCrypt加密）、登录（JWT、限流锁定）、记住密码、个人信息管理、用户状态管理、权限控制
2. **演唱会管理模块**（仅管理员）：增删改查、状态枚举、筛选查询、模糊搜索
3. **票务管理模块**（核心高并发）：票档配置、Redis+MySQL双存储、Redis Lua原子扣减、令牌桶限流、超卖防护、库存释放（RabbitMQ延迟队列）
4. **订单管理模块**：创建/支付/取消/退款/完结全流程、订单查询、订单日志
5. **通知模块**：WebSocket实时推送、未读小红点、系统公告/个人通知、历史通知分页查询

### 技术栈

- Backend: Java 17 + Spring Boot 3.2 + MyBatis-Plus + MySQL 8.0 + Redis + RabbitMQ + WebSocket
- Frontend: 原生 HTML5 + CSS3 + JavaScript (ES6+)
- Infrastructure: Docker + Docker Compose + Nginx
